# 知识视频APP

一个基于书籍内容的知识分享移动应用，通过AI技术将书籍精华内容转化为视频，为用户提供沉浸式的知识学习体验。支持iOS和Android双平台，可发布至App Store和Google Play应用商店。

## 📱 项目简介

本项目是一个类似抖音的垂直视频移动应用，专注于知识内容的传播。用户可以通过上下滑动的方式浏览不同类别的知识视频，包括科技、艺术人文、商业业务三大领域。应用采用跨平台开发框架，一套代码同时支持iOS和Android平台。

## 🔄 内容上传流程

```
书籍录入 → AI提取精华 → AI软件生成视频 → 人工审核 → 发布上线
```

### 详细流程说明

1. **书籍录入**
   - 后台管理员录入书籍基本信息（书名、作者、ISBN等）
   - 上传书籍电子版或指定章节范围

2. **AI提取精华**
   - 通过AI软件（如GPT、Claude等）分析书籍内容
   - 提取核心观点、关键概念、重要案例
   - 生成视频脚本大纲

3. **视频制作**
   - 基于AI提取的内容生成视频脚本
   - 使用AI视频生成工具（如Sora、Runway等）或人工制作
   - 添加字幕、配乐、转场效果

4. **人工审核**
   - 内容准确性审核
   - 视频质量检查
   - 版权合规性确认
   - 分类标签确认

5. **发布上线**
   - 设置发布时间
   - 选择分类标签（科技/艺术人文/商业业务）
   - 配置封面图和描述
   - 发布到对应分类

## 💾 服务器和数据库需求说明

### ✅ 为什么需要服务器和数据库？

根据项目需求，**必须要有服务器和数据库**，原因如下：

#### 1. **核心功能依赖**
- **视频内容管理**: 视频文件、元数据、分类信息需要存储和管理
- **用户数据**: 用户信息、登录状态、个人偏好需要持久化存储
- **交互数据**: 点赞、评论、收藏、播放历史等需要实时记录和查询
- **内容审核**: 审核状态、审核历史需要追踪和管理

#### 2. **业务逻辑处理**
- **AI内容提取**: 需要服务器调用AI API处理书籍内容
- **视频处理**: 视频上传、转码、存储需要服务器端处理
- **内容分发**: 视频CDN配置、分类筛选、推荐算法需要服务器支持
- **权限控制**: 用户权限、内容访问控制需要服务器验证

#### 3. **数据安全**
- **API安全**: 接口鉴权、防刷、限流需要服务器控制
- **数据加密**: 敏感信息加密存储和传输
- **内容审核**: 防止违规内容发布

### 📈 不同用户量级推荐方案

#### 300人用户量（推荐方案）

**用户特征分析**:
- 总用户数: 300人
- 预估日活: 60-90人（20-30%日活率）
- 预估并发: 10-20人同时在线
- **内容发布**: 每天10条视频，每条1分钟
- **视频大小**: 1分钟视频约15-25MB（720p-1080p，压缩后）
  - 每天新增: 10条 × 20MB = 200MB/天
  - 每月新增: 200MB × 30 = 6GB/月
  - 一年累计: 约72GB
- **视频播放流量**: 假设每人每天观看10个视频
  - 日流量: 75人（日活中值）× 10个 × 20MB = 15GB/天
  - 月流量: 15GB × 30 = 450GB/月

**推荐方案：BaaS平台（LeanCloud 或 Supabase）**

**为什么选择BaaS**:
- ✅ **成本低**: 免费套餐通常足够300人使用
- ✅ **无需运维**: 无需管理服务器和数据库
- ✅ **开发快速**: 内置认证、存储、实时功能
- ✅ **自动扩展**: 用户增长时无需担心扩容
- ✅ **稳定可靠**: 大厂维护，稳定性好

## 安全特性

### 用户访问控制
- 🔒 **仅限预注册用户**: 只有通过后台管理创建的用户才能使用邮箱OTP登录
- 🚫 **阻止未授权访问**: 未注册用户尝试登录时会收到"用户不存在，请联系管理员注册账号"的提示
- 👥 **管理员管理**: 所有用户账号由管理员统一管理，确保安全性

### 安全流程
1. **管理员创建用户**: 在后台管理界面添加用户账号
2. **用户获取OTP**: 只有已注册用户才能请求OTP验证码
3. **OTP验证登录**: 使用6位数字验证码完成登录
4. **会话管理**: 安全的token-based认证系统
